# 三井住友銀行 FOX 外為取引システム近代化

この案件は、単なる外為システム開発ではありません。高信頼性が求められる銀行コアシステムを、逆解析・移行・自動化・災対運用の観点で立て直した案件です。70～90台規模のサーバーと約9TBのOracle基盤を対象とした更改プロジェクトでした。

## 営業向け要約

- 顧客: 三井住友銀行
- システム: FOX 外為 / 金融派生商品取引システム
- 期間: 2023-06 ～ 2024-01
- 所属: DXC Technology
- 規模: 70～90台サーバー、Oracle 約9TB、ファイルデータ約4.1TB
- 移行先: HP-UX 11.3 / Itanium IA64 から RHEL 8.2 x86_64

営業向けには、まず「銀行コア」「大規模移行」「災対自動化」「既存資産解析」の4点で見せるのが分かりやすいです。

## 面接用 30秒説明

三井住友銀行の FOX 外為・金融派生商品システム更改案件です。

20年以上運用されている HP-UX / Itanium 基盤を RHEL 8 へ移行するプロジェクトで、既存 Shell・Oracle・IF 制御を解析しながら、DR 切替自動化、移行検証ツール開発、データ移行検証を担当しました。

特に、設計書不足環境での既存資産解析とリスク低減が主な役割でした。

## 何が難しかったか

- 20年以上かけて積み上がった基幹ロジック
- 設計書が不足、または実態と一致していない
- Shell スクリプトやコンパイル済みバイナリしか残っていない箇所がある
- 銀行業務のため、失敗時の影響が大きい
- 移行時も可用性、IF、DR を壊せない

## 主な担当領域

### 1. 既存資産解析・逆解析

Shell スクリプト、バイナリ、運用手順をもとに、システムの実動作を復元しました。

- `Ext_ScnCng.sh`
- `OpeSetDrExtlNode.sh`
- `SysSetScnStsDb`

DR 切替が単なる hosts 変更ではなく、Oracle の設定表や制御表の更新を伴うことを突き止め、後続の自動化方針につなげました。

### 技術的に一番難しかった点

最も難しかったのは、DR 切替処理の実際の動作が設計書に存在しなかったことです。

Shell スクリプト、Oracle 制御表、FTP 制御表、複数の運用手順を逆解析し、

「どのパラメータが実際に切替を制御しているのか」

を特定しました。

その結果、DR 切替作業を安全に再実行可能な形へ整理し、後続の自動化にもつなげました。

### 2. Oracle 移行・容量計画

- Oracle 11g / 19c / 21c の互換性確認
- CDB / PDB 対応の論点整理
- 約9TB規模の Oracle 基盤に対し、容量逼迫リスクを洗い出し
- 移行後の拡張計画策定を支援
- 89% 近い利用率のコア領域を含む拡張検討

### 3. 自動化スクリプト再構築

125 Base / 125 DB インスタンスを対象に、パラメータ駆動の Bash 自動化へ整理しました。

- ユーザー作成
- Schema 作成
- Oracle 初期化
- 反復作業の一括実行

### 4. データ移行検証

Java / Spring Batch により移行検証ツールを開発しました。

- 数百万件規模のデータ比較を自動化
- 旧新比較
- 差異検出の自動化
- 手作業工数を約30%削減

従来は手作業中心だった検証工程を効率化し、移行テスト工数を約30%削減しました。

### 5. 災対（DR）自動化

高リスクな DR 切替を、安全に再実行しやすい形へ整理しました。

- DRLiveRun
- Standby / Active の動作整理
- Oracle 制御表
- FTP 制御表

### 6. インタフェース統制

外部 IF は約61本ありました。

- SWIFT
- HULFT
- MQ
- OBS
- SFTP
- SSH

Host 切替で済むもの、DB パラメータで切り替えるもの、個別のプロトコル処理が必要なものを切り分けました。

### 7. セキュリティ・ミドルウェア評価

Tuxedo の暗号化評価では、TLS、暗号化オーバーヘッド、高並列時の性能バランスを確認しました。

## この案件の価値

この案件の価値は、Shell や Oracle そのものではなく、次の能力です。

- 既存資産解析
- 大規模移行
- 災対自動化
- 銀行コアのリスク低減
- ドキュメント欠落環境での実務推進

この案件では単なる移行作業ではなく、

- 設計書不足環境での既存資産解析
- 大規模基盤移行
- Oracle 更改
- DR 自動化
- 移行品質保証

を横断的に担当しました。
