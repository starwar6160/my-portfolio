## 1. トヨタ向け AI 音声営業アシスタント
**期間:** 2022.11 - 2023.01

営業会話をリアルタイムで支援する AI システムを設計しました。
通話中の顧客発話を解析し、以下のような重要シグナルを営業担当へ提示しました。

- 予算
- 家族構成
- 希望車種
- ローン意向

### システム構成

- フロントエンド: PowerApps / Flutter
- API: WebSocket
- バックエンド: Lambda (Python)
- 音声処理: Azure Speech SDK
- 形態素解析: MeCab
- テキスト解析: Azure Text Analytics
- 保存先: RDS / S3
- 出力: フロントエンドへのリアルタイム表示

### 主な技術

- AWS Lambda
- Azure Speech SDK
- Azure Text Analytics
- Python
- WebSocket
- MeCab
- RDS

### 工学的な難所

- リアルタイム音声ストリーミング
- 話者分離
- サービス間のタイムスタンプ整合
- Lambda のライフサイクル復旧
- 複数サービスのオーケストレーション

### 技術ポイント

- 音声認識と自然言語解析をストリーミング処理で統合
- Azure Speech で文字起こしし、AWS Transcribe で話者分離するハイブリッド構成を採用
- 複数の音声サービス結果をタイムスタンプベースで統合
- 100ns 単位の Azure 時刻と秒単位の AWS 時刻を正規化
- 顧客発話と営業発話を分離
- Lambda のライフサイクル不安定性に対してチェックポイント復旧を設計
- 高頻度の音声ストリームと低頻度の業務ロジックを分離
- Lambda 内で WebM 音声チャンクを再構成し、連続したストリームとして認識可能な形式へ変換
- 複数 Lambda の協調処理のためのメッセージプロトコルを定義
- 集約結果を RDS に保存

### 成果

営業担当は、顧客と会話しながらタブレット上でリアルタイムの指標を確認できるようになりました。
リアルタイム営業支援のための AI Copilot として機能しました。

## 2. n8n + LLM ワークフローの再設計
**期間:** 2025 Q3

予測ワークフローを、保守性、可観測性、コスト制御の観点から再設計しました。

### 技術内容

- 大きなワークフローを小さな段階に分割
- 実行時間を数時間から数分へ短縮
- Token 使用量と運用コストを削減
- 可観測性と保守性を改善
- 複雑な業務ロジックをバックエンドサービスへ移動して拡張性を向上
- 繰り返し処理を自動化

### アーキテクチャの要点

- 直列の 1 件ずつ処理ではなく、段階的なバッチ実行へ変更
- モデル呼び出し前にバックエンド側で payload の軽量化と検証を実施
- ワークフローは重い業務ロジックではなくオーケストレーションに集中
- 例外系は早期に検知して下流への影響を抑制

### 技術的な効果

実行時間は約 3 時間から約 8 分へ短縮され、運用コストは 1 実行あたり約 $0.13 まで圧縮されました。
また、運用しやすく、デバッグしやすく、拡張しやすい構成になりました。

## 3. LLM 駆動の大規模システム開発
**期間:** 2025.08 - 2026.01

LLM ツールを活用し、バックエンド、Web、ネットワーク、クラウドを横断する大規模システムの理解と実装を加速しました。

### 主な作業

- 未知のコードベースの理解を高速化
- 大きなタスクを実行可能な工程へ分解
- バックエンド、フロントエンド、インフラをまたぐ障害切り分け
- LLM ベースのツールでコード理解と実装を加速
- 複数案件の並行時にもシステム文脈を素早く回復
- AI は設計判断の代替ではなく、エンジニアリングの加速装置として使用

### 利用ツール

- Claude Code
- OpenAI モデル
- Kilo Code
- GLM

### 実務上の立ち位置

私は主にシニアバックエンドエンジニアとして動き、必要に応じて AI ツールを使って実装、説明、学習を加速しました。
実務では、バックエンドだけでなく、必要なフロントエンドや連携部分もまとめて対応できる形でした。

## まとめ

私の AI 経験は、本番向け AI システムと AI を活用したソフトウェア開発に重点があります。

共通する技術パターンは次の通りです。

- AI サービス、クラウド基盤、リアルタイムシステムの統合
- 音声、テキスト、ワークフロー、バックエンドの連携
- 信頼性、遅延、保守性の課題解決
- 本番グレードのシステム提供
- AI をソフトウェア開発の force multiplier として活用
