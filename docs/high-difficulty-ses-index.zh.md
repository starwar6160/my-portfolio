# 高难度技术实绩索引（中文参考）

这是给自己看的中文参考，不放进正式日文首页。

## 优先级 A

1. SM9 低层实现
   - 8KB RAM、ASN.1、TRNG、JNI、驱动联动
   - 适合低资源加密、硬件直连、驱动联调难题

2. 12KB 的战壕
   - 12KB RAM、`MR_STATIC`、`mrkill`、MicroLib、极限资源控制
   - 适合裸机、低内存、加密实现、系统主权类表达

3. USB HID 协议
   - 64 字节限制、Report ID、65 字节绕过、Windows 驱动兼容性
   - 适合协议边界、设备控制、Windows 周边问题

4. VC6 / ABI 兼容性
   - VC6/VS 混编、`LNK2005`、`__cdecl` / `__stdcall`、静态链接
   - 适合老银行系统、ABI 冲突、遗留环境延寿

5. 跨语言边界防崩设计
   - C++ × C# / Java、heap corruption、GC、调用约定、JNI / SWIG / JNA
   - 适合 DLL 边界、跨语言集成、内存破坏分析

6. 离线要塞化
   - Linux 内核裁剪、攻击面最小化、air-gapped、启动优化
   - 适合安全基建、闭网环境、保守型现场

7. 3DES / AES 遗留救援
   - 3DES 到 AES-GCM 的非破坏迁移、密钥轮换、旧硬件延寿
   - 适合金融、工业控制、无法整体替换的遗留系统

## 优先级 B

8. 技术债治理
   - 模块化单体、ETL 提速 50 倍、P0 缺陷治理
   - 适合工程治理、性能改善、保守型改造项目

## 后续分组建议

- 低层 / 嵌入式
- 遗留救援 / 兼容性
- 语言边界 / 驱动
- 安全基础设施
