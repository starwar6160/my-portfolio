# 站点定位说明

## 核心定位
这个站点对外展示的是“复杂系统可信工程师”画像，不是普通 SES 招聘页。对外口径要稳定落在以下方向：

- Complex systems
- Legacy modernization
- Infra / Platform / SRE
- Embedded / Automotive
- Reliability / Troubleshooting
- Protocol / boundary / migration work

## 页面分层
- 首页：企业友好，强调能力边界和技术可信度。
- `portfolio-ses`：技术档案页，给营业和技术负责人快速判断适配性。
- `docs/high-difficulty-ses-index`：实绩索引页，只做案例筛选，不做夸张包装。
- `docs/` 与 `cases/`：补充材料，保留具体案例和技术细节。

## 语气规则
- 优先使用 `reliability`、`stabilization`、`hardening`、`design review`、`migration`、`troubleshooting`。
- 尽量少用 `案件`、`参画`、`即日`、`救火`、`高難度`、`高性能`、`throughput` 这种容易把页面拉回派遣广告感的词。
- 不改对外 URL，只改标题、摘要和正文语气。

## 维护原则
- 保留企业技术岗位可信度。
- 保留营业可读性，但不把页面写成纯业务单。
- 让页面先像技术品牌，再像履历。

## 更新建议
每次改站点时，先检查标题是否统一，再检查页面是否仍然强调复杂系统适配，而不是单纯的单点性能或单价叙事。
